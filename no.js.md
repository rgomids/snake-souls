# no.js.md - Regras Complementares

## Changelog obrigatório

- O projeto deve manter um arquivo `CHANGELOG.md` na raiz.
- Toda alteração funcional, visual, estrutural ou de documentação relevante deve gerar atualização imediata no `CHANGELOG.md`.
- Cada entrada do changelog deve incluir:
  - data (`YYYY-MM-DD`);
  - versão do jogo;
  - resumo objetivo do que foi alterado.
- Nenhuma entrega é considerada concluída sem registrar a alteração no changelog.
